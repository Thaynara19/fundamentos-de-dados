## 🧠 Aprendizado 1 — Ciclo de Vida dos Dados

Estudei o conceito de **Ciclo de Vida dos Dados**, que representa todas as etapas pelas quais os dados passam dentro de uma organização.

### Etapas do ciclo de vida:

1. **Planejar** — Definir quais dados são necessários.
2. **Capturar** — Coletar os dados.
3. **Gerenciar** — Armazenar e manter os dados organizados.
4. **Analisar** — Explorar e interpretar os dados.
5. **Arquivar** — Guardar dados que ainda possuem valor.
6. **Destruir** — Remover dados que não são mais necessários.

💡 Entendi que os dados possuem um ciclo completo dentro das empresas, desde a criação até o descarte seguro.

---

## 🧠 Aprendizado 2 — Processo de Análise de Dados

Aprendi também o **Processo de Análise de Dados**, utilizado por analistas para resolver problemas de negócio.

### Etapas do processo:

1. **Perguntar (Ask)** — Definir o problema e entender expectativas.
2. **Preparar (Prepare)** — Coletar dados relevantes.
3. **Processar (Process)** — Limpar e organizar dados.
4. **Analisar (Analyze)** — Encontrar padrões e respostas.
5. **Compartilhar (Share)** — Comunicar resultados.
6. **Agir (Act)** — Apoiar tomadas de decisão.

Durante a atividade prática, compreendi que definir o problema faz parte da etapa **Perguntar**, e não da análise em si.

---

## 🔎 Diferença importante aprendida

- O **Ciclo de Vida dos Dados** mostra como os dados existem dentro da organização.
- O **Processo de Análise de Dados** mostra como o analista trabalha para resolver problemas usando dados.

Percebi que a análise de dados começa muito antes dos gráficos ou consultas SQL.

Primeiro é necessário compreender o problema, o contexto do negócio e como os dados são gerenciados ao longo do tempo.

Esse aprendizado fortaleceu minha base para atuar futuramente como Analista de Dados.
